# Sprint Skill Library

Run the relevant skill before touching any subsystem listed below. Full prompts, diagnostic tables, and code snippets are in `Docs/skills/`.

|---|---|---|---|

| 2 | Binary Diagnostic Probe | KLV walk produces garbage FourCCs, wrong sample count, or unexpected `ErrMalformedGPMF` | [Docs/skills/skill-2-binary-diagnostic.md](Docs/skills/skill-2-binary-diagnostic.md) |
| 3 | Manual BigEndian Audit | Before every PR that modifies the Go parser | [Docs/skills/skill-3-bigendian-audit.md](Docs/skills/skill-3-bigendian-audit.md) |

| 5 | Write-Through Cache Flow | Before implementing or debugging WASM parsing or backend API calls | [Docs/skills/skill-5-write-through-cache.md](Docs/skills/skill-5-write-through-cache.md) |

| 10 | Canvas Alpha Guard | Before adding any new Canvas primitive that uses `globalAlpha` or transparency. Checklist: (1) Is the draw wrapped in `ctx.save()`/`ctx.restore()`? (2) Is `backgroundAlpha` sourced from `ThemeConfig.map` — not hardcoded? (3) Does the hex colour optically mix acceptably at the chosen alpha over dark video? Light colours below 0.5 alpha over dark video = muddy gray. | [Docs/skills/skill-10-canvas-alpha-guard.md](Docs/skills/skill-10-canvas-alpha-guard.md) |
| 11 | Strava Biometrics Checklist | Before modifying `drawBiometrics()`, `interpolateBiometrics()`, adding new GPX fields, changing HR zone thresholds, or adding a new layout variant that must render bio data. | [Docs/skills/skill-11-strava-biometrics.md](Docs/skills/skill-11-strava-biometrics.md) |
| 12 | Biometric Data Handling | Before writing any new code that reads or interpolates `StravaGpsPoint[]`. Checklist: (1) Is `syncOffsetMs()` added to the time argument before every `interpolateBiometrics()` call? The function is a pure binary search — the caller owns time adjustment. (2) Are you mutating `StravaGpsPoint.t` or `.relativeTimeSec`? **Stop** — those are anchor-time constants; the offset shifts the lookup, not the data. (3) Does the new path exist in both the live HUD (`drawFrame`) and the export loop? Both sites must apply the offset independently. (4) Is `syncOffsetMs.set(0)` called at the top of any method that resets session state (new video load, new GPX upload)? | [Docs/skills/skill-12-biometric-sync.md](Docs/skills/skill-12-biometric-sync.md) |
| 13 | Path2D Cache Key Audit | Before any change to `mapZoom`, the Path2D cache invalidation condition, or any field in `cacheKey`. Checklist: (1) Is the cache storing `cacheZoom` (not raw `zoom`)? `cacheZoom = zoom <= 0 ? Math.round(zoom) : 1`. (2) Does every bbox sentinel branch (`-2`, `-1`, `0`) read `cacheZoom`, not `zoom`? A fractional slider value like `-0.1` must round to `0`, not fall through to the wrong bbox. (3) Are the double-stroke pairs each wrapped in their own `save()`/`restore()`? Sharing one pair across outline + primary makes it impossible to set different `globalAlpha` on ghost vs. segment strokes. (4) Is the max `save()` nesting depth still ≤ 2 (outer clip/transform + one stroke block)? | — |
